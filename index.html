dyna4jdbc

A JDBC driver implementation that allows running dynamic JVM languages (Scala, JavaScript, Groovy, Clojure, Python, R language /via Renjin JVM interpreter/, JRuby etc.) or external console-oriented programs (written in shell, PERL etc.) via the JDBC API.

Use Cases Learn More

Start Immediately

Add the driver and the dynamic language libraries to the classpath of your JDBC-reporting tool.

Implement

Write your Scala, JavaScript, Groovy, Clojure, Python, R, JRuby, BeanShell etc. script to gather and analyze data.

Generate the result

Simply output the results directly from your script. The engine takes care of the conversion.

Build reports

Use your well known and powerful JDBC-enabled reporting tools to analyze and visualize the results.

Use cases

Where can you use it?

Lightweight reports

Create reports from your JDBC reporting tools without loading your data into a relational database.

Data from the Cloud

Retrieve report data from all kinds of Cloud services. The reporting tool will see your results as a JDBC Result Set.

Advanced Analysis

Fetch and process data using your favourite scripting language, directly from your well known JDBC application.

Endless Potential

Use it as a building block for your systems. Include it in your application, provide additional features on top of it.

Simple and powerful

You write the whole logic for your reports: simply echo the results to the output of the host script language.

Structuring

Echo a tabulator ( \t ) to separate cells (columns); a newline ( \n ) starts a new row in the result set.

Data types

Column types are either detected automatically or specified by the first row emitted by the script.
